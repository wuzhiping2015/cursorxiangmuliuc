```plaintext
# MacOS群发系统项目功能需求PRD

## 一、项目概述
本项目旨在开发一款运行于MacOS系统的群发系统，方便用户进行消息的批量发送。系统将具备用户管理、消息管理、群发设置和发送记录查看等核心功能。

## 二、功能需求

### （一）用户管理
1. **用户添加**
    - 提供添加用户的界面，允许用户输入用户名、邮箱等信息。
    - 点击"添加用户"按钮后，系统将用户信息保存到数据库。
2. **用户编辑**
    - 在用户列表中，为每个用户提供"编辑"按钮。
    - 点击"编辑"按钮，弹出编辑窗口，用户可修改用户名、邮箱等信息。
    - 修改完成后，点击"保存"按钮，系统更新数据库中的用户信息。
3. **用户删除**
    - 在用户列表中，为每个用户提供"删除"按钮。
    - 点击"删除"按钮，弹出确认对话框，确认后系统从数据库中删除该用户信息。
4. **用户列表展示**
    - 以表格形式展示所有用户信息，包括用户名、邮箱等。

### （二）消息管理
1. **消息撰写**
    - 提供文本编辑区域，用户可输入消息内容。
2. **消息保存**
    - 点击"保存"按钮，系统将消息内容保存到数据库。
3. **消息删除**
    - 点击"删除"按钮，弹出确认对话框，确认后系统从数据库中删除该消息。

### （三）群发设置
1. **发送对象选择**
    - 提供下拉菜单，用户可选择"全部用户"或特定的用户组。
2. **发送时间设置**
    - 提供日期和时间选择器，用户可设置消息的发送时间。
3. **消息发送**
    - 点击"发送"按钮，系统根据用户选择的发送对象和发送时间进行消息群发。

### （四）发送记录查看
1. **记录列表展示**
    - 以表格形式展示所有已发送消息的记录，包括消息内容、发送时间、发送状态等。

## 三、非功能需求
1. **界面设计**
    - 采用MacOS简洁的设计风格，使用浅灰色背景，白色内容区域，深色文字。
    - 按钮和图标使用MacOS常见的样式，如圆角矩形按钮。
2. **性能要求**
    - 系统响应时间应控制在合理范围内，避免出现长时间的卡顿。
3. **安全性要求**
    - 对用户信息和消息内容进行加密存储，防止数据泄露。
```

# MacOS群发系统技术栈选择与技术协作规划

## 一、技术栈选择

### （一）前端
1. **框架**：Vue 3
    - 原因：具备高效的响应式系统和组合式API，能提升开发效率和代码可维护性。
    - JAVASCRIPT: 开发简单
2. **组件库**：Element Plus（适用于Vue 3的Element UI）
    - 原因：提供丰富且美观的组件，可快速搭建界面。
3. **图标库**：FontAwesome
    - 原因：提供大量美观的图标，可增强界面的视觉效果。
     - 真实感增强：
     - 界面尺寸应模拟iPhone 15 Pro，并让界面圆角化，使其更像真实的手机界面。
     - 使用真实的UI图片，而非占位符图片（可从Unsplash、Pexels、Apple官方UI资源中选择）。
     - 添加顶部状态栏（模拟iOS状态栏），并包含App导航栏（类似iOS底部Tab Bar）。
### （二）后端
1. **语言**：Python
    - 原因：语法简洁，有丰富的库和框架可供使用。
2. **框架**：Flask
    - 原因：轻量级框架，易于上手和快速开发。
3. **数据库**：SQLite
    - 原因：嵌入式数据库，无需额外的服务器配置，适合小型项目。

### （三）部署
1. **打包工具**：PyInstaller
    - 原因：可将Python程序打包成独立的可执行文件，方便在MacOS系统上部署。

## 二、技术协作
1. **前端与后端协作**
    - 前端通过API与后端进行数据交互，使用JSON格式传输数据。
    - 后端提供RESTful API接口，前端根据接口文档进行调用。
2. **数据库与后端协作**
    - 后端使用SQLAlchemy库与SQLite数据库进行交互，实现数据的增删改查操作。

### 软件开发工程师 - 开发指令
```plaintext
# MacOS群发系统开发指令

## 一、项目初始化
1. 创建项目目录，命名为 `macos-mass-sender`。
2. 初始化前端项目：
    - 进入项目目录，执行 `npm init vite@latest client -- --template vue` 命令创建Vue 3项目。
    - 进入 `client` 目录，执行 `npm install element-plus @element-plus/icons-vue font-awesome crypto-js jsencrypt axios-retry` 命令安装所需依赖。
    - 在 `main.js` 中引入并配置Element Plus：
```javascript
import { createApp } from 'vue'
import App from './App.vue'
import ElementPlus from 'element-plus'
import 'element-plus/dist/index.css'

const app = createApp(App)
app.use(ElementPlus)
app.mount('#app')
```
3. 初始化后端项目：
    - 在项目根目录下创建 `server` 目录。
    - 进入 `server` 目录，执行 `python -m venv venv` 命令创建虚拟环境。
    - 激活虚拟环境：在MacOS系统上执行 `source venv/bin/activate` 命令。
    - 执行 `pip install flask sqlalchemy` 命令安装所需依赖。

## 二、后端开发

### （一）数据库模型设计
1. 在 `server` 目录下创建 `models.py` 文件，定义用户、消息和发送记录的数据库模型。
2. 使用SQLAlchemy库与SQLite数据库进行交互。

### （二）API接口开发
1. 在 `server` 目录下创建 `app.py` 文件，使用Flask框架开发RESTful API接口。
2. 实现用户管理、消息管理、群发设置和发送记录查看等功能的API接口。

## 三、前端开发

### （一）界面设计
1. 使用Vue 3和Element Plus实现用户管理、消息管理、群发设置和发送记录查看等界面。
2. 使用FontAwesome图标库增强界面的视觉效果。

### （二）与后端交互
1. 使用 `axios` 库与后端API接口进行数据交互。
2. 实现用户添加、编辑、删除，消息保存、删除，群发设置和发送记录查看等功能。

## 四、项目测试
1. 编写单元测试和集成测试代码，确保各个功能模块正常运行。
2. 进行手动测试，验证系统的整体功能和用户体验。

## 五、项目部署
1. 使用PyInstaller将后端Python程序打包成独立的可执行文件。
2. 将前端项目进行打包，生成静态文件。
3. 将打包后的后端可执行文件和前端静态文件部署到MacOS系统上。
```
